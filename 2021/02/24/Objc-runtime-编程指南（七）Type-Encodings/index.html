Type Encodings

为了帮助运行时系统，编译器会将将每个方法的参数和返回值编译成一个c的字符串与方法选择器的字符串关联。这种编译器使用编码方案在其他环境中也很适用，因此可以通过 @encode() 编译器指令来公开使用。对于给定的一个特定类型, @encode() 会返回一个string来编码这个类型。 这个类型可以是像 int 的基础类型，可以是指针，也可以是带标记的结构或者联合体，又或者是一个类名 —— 任何类型！ 事实上，是可以被 C sizeof() 操作符作用的所有类型。

下面的表格列举了类型编码。请注意，它们中的许多与您出于归档或分发目的而对对象进行编码时使用的代码重叠。然而（还需要注意），这儿有一些编码是你在写编码器时不能使用的；还有一些你在编写一个编码器想使用的，但不是由 @encode() 的编码。（在归档/分发时编码对象时，请参见 Foundation 框架参考的 NSCoder 类声明获取更纤细的信息。）

Table 6-1 Objective-C type encodings

Code Meaning
c A char
i An int
s A short
l A long
q A long long
C An unsigned char
I An unsigned int
S An unsigned short
L An unsigned long
Q An unsigned long long
f A float
d A double
B A C++ bool or a C99 _Bool
v A void
* A character string (char *)
@ An object (whether statically typed or typed id)
# A class object (Class)
: A method selector (SEL)
[array type] An array
{name=type…} An structure
(name=type…) A union
bnum A bit field of num bits
^type A pointer to type
? An unknown type (among other things, this code is used for function pointers)
